# Functional Requirement - Send Email Notification

## 1. Feature Overview

Send email notification for critical/system events through configured email provider.

## 2. Actors

- **Notification Worker:** Renders and sends email.
- **Email Provider:** Delivers email.
- **User:** Receives email.

## 3. Scope

**In Scope:**

- Validate email channel policy.
- Render server-side template.
- Send email via provider.
- Classify provider failure.

**Out of Scope:**

- Marketing email.
- Email template builder.
- Generating Auth verification/reset tokens.

## 4. Trigger

- Processing a notification event whose channel policy includes `EMAIL`.

## 5. Business Rules

- Email is used only for critical/system events in MVP.
- User setting `allow_email` is respected unless critical override applies.
- Template variables must be sanitized.
- Notification Service delivers tokens/links generated by owner service; it does not generate credential tokens.

## 6. Database Impact

- Read `notification_events`.
- Read `user_notification_settings`.
- Update delivery status/retry metadata according to implementation.

## 7. Transaction

- Do not hold long DB transaction while calling provider.

## 8. Security

- Never log OTP, reset token, provider credentials or raw email auth headers.

## 9. Failure Cases

- Provider timeout/rate limit -> retryable.
- Invalid email -> permanent failure.
- Missing template -> failed.
- Missing required template variable -> failed.

## 10. Acceptance Criteria

- Email is sent for eligible events.
- Non-critical events do not send email by default.
- Provider failures are classified.
- Sensitive values are not logged.
