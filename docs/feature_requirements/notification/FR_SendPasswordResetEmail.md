# Functional Requirement - Send Password Reset Email

## 1. Feature Overview

Send password reset email when Auth Service publishes `PASSWORD_RESET_REQUESTED`.

## 2. Actors

- **Auth Service:** Owns reset token and validation.
- **Notification Service:** Sends reset email.
- **User:** Receives reset link/code.

## 3. Scope

**In Scope:**

- Validate reset email payload.
- Render password reset template.
- Send email via provider.

**Out of Scope:**

- Creating reset token.
- Resetting password.
- Revoking sessions after password reset.

## 4. Trigger

- Event type `PASSWORD_RESET_REQUESTED`.

## 5. Business Rules

- Payload must include recipient email and reset link/code generated by Auth.
- Reset email is security-critical.
- Notification Service must not expose whether account exists beyond Auth's public behavior.
- Reset token/link must not be logged or stored in `last_error`.

## 6. Database Impact

- Read/update `notification_events`.
- Optional delivery status update.

## 7. Failure Cases

- Missing reset link -> failed.
- Provider timeout -> retry.
- Invalid email -> permanent failure.

## 8. Security

- Never log reset token/link.
- Use sanitized provider errors.

## 9. Acceptance Criteria

- Valid reset event sends reset email.
- Auth remains token owner.
- Failure is retryable/permanent by provider error type.
