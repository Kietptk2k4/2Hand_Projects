# Functional Requirement - Send Email Verification Email

## 1. Feature Overview

Send email verification message when Auth Service publishes `EMAIL_VERIFICATION_REQUESTED`.

## 2. Actors

- **Auth Service:** Publishes verification event and owns token.
- **Notification Service:** Sends email.
- **User:** Receives verification link/code.

## 3. Scope

**In Scope:**

- Consume verification event.
- Render verification template.
- Deliver email.

**Out of Scope:**

- Generating or validating verification token.
- Updating Auth user verification status.

## 4. Trigger

- Event type `EMAIL_VERIFICATION_REQUESTED` from Auth Service.

## 5. Business Rules

- Payload must include recipient email and verification link/code generated by Auth.
- Notification Service must not create verification token.
- Email is security-critical and can be sent regardless of normal marketing preferences.
- Token/link must not be logged.

## 6. Database Impact

- Read/update `notification_events`.
- Optional delivery status update.

## 7. Security

- Do not store raw verification token in logs.
- Email template must avoid exposing internal user state.

## 8. Failure Cases

- Missing email -> failed.
- Missing verification link/code -> failed.
- Provider transient failure -> retry.
- Invalid email -> permanent failure.

## 9. Acceptance Criteria

- Verification email is sent when event is valid.
- Auth remains owner of verification token.
- Sensitive token/link is protected in logs.
